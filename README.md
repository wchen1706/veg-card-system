## 📖 项目简介
本项目是一个专为生鲜零售门店定制的 **会员预付费卡片管理与运营系统**。
针对传统线下生鲜门店在“预付费卡扣减、多卡并行、欠费追踪、财务对账”等环节中的痛点，提供了一套轻量级、高可用、数据可追溯的数字化解决方案。系统大幅降低了人工记账的错误率，提升了门店的运营效率。

## ✨ 核心业务模块
本系统不仅实现了基础的 CRUD 操作，更深入贴合了真实的生鲜配送与门店运营逻辑：

* **💳 灵活的会员与卡券引擎**
    * 支持多种规格（如6斤/8斤）与周期（月卡/季卡/年卡）的开卡组合。
    * 同一个会员名下支持多张卡片并行运转，系统自动识别并管理生命周期。
* **🛒 智能扣除与跨卡防溢出逻辑**
    * **单点/批量扣款**：支持 Excel 格式直接粘贴解析，一键批量处理当日配送扣款。
    * **跨卡自动抵扣**：当主卡余额不足时，智能检索该会员名下其他有效备用卡并无缝接力扣除。
    * **异常拦截预警**：精准计算扣款后余额，产生欠费时立即触发红色预警，辅助店长催缴。
* **⚖️ 严谨的财务容错与防篡改机制**
    * 禁止直接删除或修改历史流水，保障财务数据的绝对干净与可追溯。
    * 提供正规的 **“退补调账”** 功能，所有错扣、漏扣、坏单损耗均通过生成冲销记录来修正。
* **📊 经营数据看板**
    * 自动化汇总每日发货总量与订单数，实时呈现财务流水明细，支持多维度按日期检索。

## 🛠️ 技术栈与架构设计
* **前端交互**：基于 `Streamlit` 构建响应式 Web 界面，深度定制 UI 样式，适配多端操作。
* **后端逻辑**：`Python`，封装了复杂的业务路由与容错校验机制。
* **数据存储**：`PostgreSQL` (托管于 Supabase)，采用多表联查与外键级联设计，保证数据一致性。
* **数据分析**：`Pandas`，用于高速解析批量粘贴的文本数据及构建数据看板。
* **部署方案**：`Streamlit Cloud` 持续集成部署，结合 Git 分支策略（`main` 为生产环境，`demo` 为展示环境）。

## 📂 数据库核心 ER 模型 (简述)
* `members`：会员主表（唯一手机号鉴权）。
* `cards`：菜卡明细表（记录规格、周期、历史总额度与当前余额）。
* `records`：操作流水表（记录每一笔发货、调账、散客零售的去向与操作人）。
